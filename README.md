# Bus Bunching Prediction via Early Warning Signals in a Transit Digital Twin

GTFS-RT data pipeline and predictive framework for bus bunching early warning signals, using San Francisco Muni data. Submitted to **PeerJ Computer Science**.


## Overview

This project implements an end-to-end pipeline that:

1. **Collects** real-time GTFS-RT feeds (VehiclePositions + TripUpdates) from the 511.org API
2. **Stores** observations in DuckDB (embedded columnar analytics)
3. **Computes** three metrics: headway deviation, delay drift, and bunching index
4. **Predicts** bus bunching transitions using a multi-stop sequential window ($n-3$) with LightGBM and temporal hold-out validation

Current reproducible results are generated by `scripts/analyze_longitudinal.py` and written to `professor_metrics.txt`.

## Project Structure


## Quick Start

### Prerequisites

- Python 3.11+
- [uv](https://github.com/astral-sh/uv) package manager
- 511.org API key ([register here](https://511.org/developers))

### Setup

```bash
# Install dependencies
uv sync --all-extras

# Configure API key
cp env.example .env
# Edit .env and set your API_KEY
```

### Collect Data

```bash
# Single snapshot
uv run python scripts/collect.py

# Continuous collection (systemd timer, every 130s)
bash scripts/install-timer.sh
```

### Reproduce Paper Results

```bash
# 1. Download historical dataset range
uv run python scripts/download_longitudinal.py --start-month 2026-01 --end-month 2026-03

# 2. Ingest into longitudinal DuckDB view
uv run python scripts/ingest_longitudinal.py

# 3. Run longitudinal analysis
uv run python scripts/analyze_longitudinal.py

# 4. Generate PR curves
uv run python scripts/generate_pr_curve.py

# 5. (Optional) Regenerate PR figure from exported curve CSV
uv run python scripts/generate_pr_curve.py
```

### Tests

```bash
uv run pytest
uv run ruff check src/ tests/
```

## Dataset

- **Source**: San Francisco Municipal Railway (SF Muni) via [511.org](https://511.org) open API
- **Period**: Longitudinal historical ingest (configured by download range; paper uses 2026-01 to 2026-03 pipeline)
- **Volume**: See `professor_metrics.txt` for the latest reproducible counts after ingestion and lag filtering
- **Routes**: 14 (Mission St), 38 (Geary Blvd), 49 (Van Ness Ave)

## API Constraints

- 511.org rate limit: 60 requests/hour
- Each cycle: 2 requests (VehiclePositions + TripUpdates)
- Polling interval: 130 seconds

## Tech Stack

- **Python 3.11+** with uv
- **DuckDB** for embedded columnar analytics
- **httpx** for HTTP, **protobuf** for GTFS-RT parsing
- **scikit-learn** for classification
- **matplotlib + seaborn** for publication figures
- **pydantic-settings** for configuration
